Functionality

Tester Percpective

welcome.html

Please surf to http://54.76.197.180:3008/welcome.html and register. Upon registering you'll be redirected to /mail.html. You can now also login using the username and password submitted on registration.

mail.html

Usage is pretty much straighforward, and works as expected. Mail addresses are used without the @bitsplease.com postfix. Just use the username without it when addressing: i.e. send emails to username and not to username@bitsplease.com.

You can also send yourself an email (or to another user, logged in another computer), and you'll see the email received in the inbox pretty instantly.

You're also welcome to disconnect your network connection and send mail. It will be actually sent when you reconnect.

Developer Percpective

Server-side

Custom http module

The server is using a custom http module, which was originally used in the previous exercise. It was modified and extended to suit this exercise's needs.

MongoDB Schema Structure

Schemas used are user and mail. Each user document has the expected fields of username, password, first/last name, age, etc., where the email address is just the username field of the user schema. Mail also has its expected fields of to/from (user document reference), subject, body, and date.

Data Types

Information sent from/to the server uses only JSON objects (except for the HTML responses, of course).

Client-side

Backbone Model Structure

The only model in use is Email, which stores a single mail data. An EmailList collection was implemented to store a single user's collection of (inbox) mails.

Socket.IO

A Websocket connection is opened against the server upon a succesful login. In addition, the server stores a user-id hashmap, which is also used to identify Socket.IO connections, and notify users upon receiving new mail.

Custom Serialization Function

A custom serializeObject() function was implemented in public/js/default.js in order to serialize complex objects, so they could be sent properly to the server using JSON requests.

Manifest (AppCache) File

A manifest file is used to cache all .html pages, in addition to all static files in use.
